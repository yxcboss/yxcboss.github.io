          # 前言

          • 作为一个前端开发，三大框架似乎是必会其中一个了，vue是其中最简单上手的，并且vue的作者为国人尤雨溪，所以对国人十分友好，从今天就开始一起学习这个热门的框架吧！

          # 什么是mvvm框架

          • View层：视图层，在前端中通常就是指DOM，主要作用就是给用户展示各种数据信息，也叫UI层；
          • Model层：数据层，通过ajax向后台接口访问回来的数据；
          • ViewModel层：视图模型层，是视图层view和数据层model沟通的桥梁，一方面实现数据的双向绑定，将Model的改变实时反应到View层；另一方面实现了dom监听，当dom发生了一些事件（点击，滚动等）时，可以监听到，并在需要时改变对应的数据

          # computed计算属性

          有时候我们可以直接使用data中的数据，但是在某些情况下，可能需要对数据进行一些转化后在显示，或者多个数据结合起来显示，就需要用到计算属性。
          基本使用
          有两个变量firstName，lastName,需要完整显示名称，有以下四种方法：
          1.直接用空格隔开

            <div>{{firstName}}  {{lastName}}</div>
          1

          2.使用加号拼接

            <div>{{firstName + ‘’ + lastName}}</div>
          1

          但是如果多个地方都需要完整显示名称的话，我们就需要写多个，看上去就很不优雅，这个时候可以把这个封装成一个方法，通过函数直接调用。 不过最好就是利用计算属性

            <div id="app">
      <!-- 方式一 -->
      <h2>{firstName}}  {{lastName}}</h2>
      <!-- 方式二 -->
      <h2>{{firstName + ‘’ + lastName}}</h2>
      <!-- 方式三 -->
      <h2>{{getfullname()}}</h2>
      <!-- 方式四 -->
      <h2>{{fullName}}</h2>
  </div>
  <script>
    export default{
      data(){
        return{
          firstName:'俞',
          lastName:'玄赐'
        }
      },
      computed:{
        getfullname(){
          return this.firstName + ' ' + this.lastName;
        }
      },
      methods:{
        fullName(){
          return this.firstName + ' ' + this.lastName;
        }
      },
    }
  </script>

          Son组件为子组件，本身为父组件，父组件需要先引入，然后在components里面注册，在到template模板中使用。
          组件对象有一个data属性。这个data属性必须是一个函数，这个函数返回一个对象，对象内部保存着数据。

          为什么data在组件中必须是一个函数呢？

          • 每个组件都是vue实例。
          • 组件共享data属性，当data的值是同一个引用类型的值时，改变其中一个会影响其他。
          • 官网说法：在组件中data必须是一个函数，这样的话，每个实例可以维护一份被返回对象的独立拷贝

          # 父子组件访问

          # 父组件访问子组件

          • $children this.$children是一个数组类型，它包含所有子组件对象，可以通过遍历，取出所有子组件信息。 注意：通过¥children访问子组件时，是一个数组类型，访问其中的子组件必须通过索引值，但当子组件过多，往往不能确定他的索引值，甚至还可能会发生变化，很多时候向明确获取其中一个特定的组件，往往使用$refs。
          • $refs
            $refs和ref指令通常是一起使用的： 1.首先，通过ref给某个子组件绑定一个特定的ID
            <Son1 ref="child1"></Son1>
  <Son2 ref="child1"></Son2>
          1
          2

          2.通过this.$refs.ID就可以访问到该组件了

            console.log(this.$refs.child1)
  console.log(this.$refs.child2)
          1
          2

          # 子组件访问父组件（不常用）

          • 子组件访问父组件通过： $parent
          • 如果访问根组件，通过： $root
            注意⚠️：真实开发中，尽量避免直接访问父组件

          # 组件高级插槽slot

          插槽就是子组件提供给父组件使用的一个占位符，用表示。父组件可以在占位符中填充任何模板代码，填充的内容会替换成子组件的标签。

          • 匿名插槽
          • 具名插槽
            子组件
            <div class="box">
    <!-- 匿名插槽 -->
    <slot /> 
    <div>
      <!-- 作用域插槽 -->
      <slot name="left" :message="message"></slot>  
    </div>
    <div>
    <!-- 具名插槽 -->
      <slot name="right"></slot>
    </div>
  </div>
