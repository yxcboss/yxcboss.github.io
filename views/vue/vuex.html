    # 一句话概括

    Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

    读了两遍确实很官方，那如何用我们自己的话去理解这个工具呢？
    vuex相当于一个仓库（store），然后放置我们所需要的的数据状态（state），通过一些方法手段（mutation，getter，action，module）去增删改查这些存放的数据
    这样子一说，vuex是不是又和全局对象好像有点像，但其实还是有区别的。

    1、Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新。

    2、你不能直接改变store中的状态。改变store中的状态唯一途径只有在mutation显式的提交（commit），这样使得我们能方便的追踪每个状的改变。 下面就单独介绍vuex里面的方法。

    # 单一状态树

    何为树：一个主枝干，然后分支出去二等枝干，二等枝干继续分支出去三等枝干。 状态树：1、一个对象（主干）包含着全部（其他分支）的层级状态。
    2、每个应用仅仅包含一个store实例对象。

    泻药，先说结果，改变store中状态的唯一方法是提交一个mutation

    # 必须是同步函数

    那为什么必须是同步函数呢？ 因为mutation来改变state状态，是为了明确追踪到状态到改变。所以官方强制规定它必须是同步函数，当然你非要写成异步的其实也行，但是会让调试变得复杂。

    # 简单介绍

    action类似于mutation，但是不同于： 1.action提交mutation事件类型； 2.action可以包含异步事件操作。

        count:0
    },
    mutation:{
        add(state){
            state.count ++

    action:{

    # 组合action

    既然是异步的，那不得不说一下Promise了，通常我们会在异步完成后在执行一些操作。那么我们可以使action函数返回一个Promise对象。

    const store = new Vuex({
    state:{
        count:0
    },
    mutation:{
        add(state){
            state.count ++
        }
    },
    action:{
        // payload接收的参数
        add({commit},payload){
            return new Promise((resolve,reject)=>{
                setTimeout(()=>{
                    commit('add');
                    resolve();
                },1000)
            })
        }
    }
})

// 派发：
this.$store.dispatch({type:'add',amount:10}).then(()=>{
    //...
})
