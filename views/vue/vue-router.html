                    # vue-router

                    vue-router即vue路由，通过vue路由的改变来显示相对应的组件。

                    # 安装vue-router

                      npm install vue-router --save
                    1

                    # 基本使用

                    1.创建路由组件

                    • 创建router实例

import VueRouter from 'vue-router'

// 注入插件
Vue.user(VueRouter)

// 定义路由
const routes=[]

// 创建router实例
const router = new VueRouter({
    routes
})

// 导出router实例
export default router

                    # 《router-link》补充 其他属性

                    • tag:制定《router-link》被渲染成什么标签
                    • replace:不会留下历史记录，回退键不能返回上一个页面
                    • active-class:会给当前选中元素设置一个router-link-active的class类，修改active-class可以修改默认的名称
                      <router-link to="/" tag="button" replace active-class="active">Home</router-link>

                    • $router作为vue-router的实例，想要导航到不同的url，则用router.push()方法;
                    • $route为当前router的跳转对象，可以获取path，name，params，query等

                    # vue-router导航守卫

                    # 什么是导航守卫

                    导航守卫主要用来监听路由的进入和离开，vue-router提供了beforeEach和afterEach的钩子函数，他们会在路由即将改变和改变后触发。

                    # 导航守卫的使用

                    例如：通过beforeEach()来完成标题的修改。 1.在路由中利用meta来定义标题

                    const routes = [
    {
      path:"/",
      component:Home,
      meta:{
        title:'首页'
      },
    },
    {
      path: '/home',
      name: 'Home',
      component: Home,
      meta:{
        title:'首页'
      },
      children:[
        {
          path:"/",
          component:Home
        },
        {
          path:'news',
          component:()=>import('../components/news')
        },

                    # 导航守卫钩子三个参数解析

                    • to:即将要进入的目标的路由对象
                    • from:当前导航即将离开的路由对象
                    • next:调用该方法后才可进入下一个钩子，进行路由跳转

                    # 导航守卫补充

                    • 如果是后置钩子（afterEach），不需要主动调用next()函数
                    • 上面使用的导航守卫，被称之为全局守卫，除此之外，还有路由独享的守卫、组件内的守卫

                    # keep-alive

                    keep-alive是Vue的内置组件，能在组件切换过程中将状态保留在内存中，防止重复渲染DOM。 keep-alive 包裹动态组件时，会缓存不活动的组件实例，而不是销毁它们。和 transition 相似， keep-alive 是一个抽象组件：它自身不会渲染一个 DOM 元素，也不会出现在父组件链中。
