            # Sass让你更快更好的书写可维护性的css

            Sass 是一个 CSS 预处理器。

            Sass 是 CSS 扩展语言，可以帮助我们减少 CSS 重复的代码，节省开发时间。

            Sass 完全兼容所有版本的 CSS。

            Sass 扩展了 CSS3，增加了规则、变量、混入、选择器、继承、内置函数等等特性。

            Sass 生成良好格式化的 CSS 代码，易于组织和维护。

            Sass 文件后缀为 .scss。 下面👇，简单介绍and记录几个我在项目中常用的sass。

            # 复用属性值————变量

            一般呢，网站，app都有属于自己的主题颜色，京东红，淘宝橙，支付宝蓝，微信绿等等。如果想要修改主题颜色还要全局搜索然后在全局替换，也许不只一种主题颜色，替换可能就会遗漏啥的，这时候sass就可以发挥他的作用了。

            $color-theme:#ff8048; //定义主题字体颜色  
$btn-color:#c00;
.txt{
    color:$color-theme;
}
.btn{
    background:$btn-color;
}

            # 复用一段样式————混合器

            在项目中，有很多样式需要重复使用，然后我们就使用cv大法（复制粘贴），比如字体溢出隐藏显示省略号，水平垂直居中等等。 有两种做法： 一、常规方法：定义一个类，然后把这个类加到所需要样式的html标签上。
            二、sass方法：定义一个选择器，以@mixin开头，后面根一个混合器名字，然后是css规则。

            @mixin note{
    overflow:hidden;
    text-overflow:ellipsis;
    white-space: nowrap;
}

            // 定义混合器
@mixin note($color){
    overflow:hidden;
    text-overflow:ellipsis;
    white-space: nowrap;
}
#notice{
    font-size:14px;
    width:200px;
    @include note(#333);
}

            # 避免重复属性————选择器继承

            一般我们会把具有重复的属性和属性值的选择器我们会写在一起，如下:

            .title,.subject{
    font-size:16px;
    font-weight:bold;
    text-align:center;
}
.title{
    color:#000;
}
.subject{
    color:#666;
}

            如上：title和subject选择器具有相同的属性和属性值，而我们有混合器的时候，就把同一段css代码添加到了不同的选择器下，造成了多余的代码，而继承更进一步的优化我们css样式的代码量,让一个选择器能够复用其他一段样式，但又不重复输出这些样式。

            .note{
    font-size:16px;
    font-weight:bold;
    text-align:center;
}
.title{
    color:#000;
    @extend .note;
}
.subject{
    color:#666;
    @extend .note;
}

            是不是符合我们日常开发的规范呢？ 但可是我不要note选择器该怎么半呢？ 这时候可以用占位符（%）来定义我们所要继承的样式。如下：

            %note{
    font-size:16px;
    font-weight:bold;
    text-align:center;
}
.title{
    color:#000;
    @extend .note;
}
.subject{
    color:#666;
    @extend .note;
