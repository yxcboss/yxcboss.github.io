            js难搞之闭包

            小俞儿 2020-05-21 闭包 JavaScript

            # 题外话

            闭包，可以说是JavaScript中很重要的一个知识点，但是对于我这种初初级前端小菜鸟来说，却没有真正的理解它，哪怕工作中也经常用到。所以该总结总结，该记记！你才会有所提升。

            # 概念

            何为闭包？ 闭包是有权访问另一个函数作用域中变量的函数
            可以这么理解：闭包就是函数，但不是一般的函数，而是可以访问另外一个函数中变量的函数。
            举个例子：有两个盒子，一个大盒子A，一个小盒子B，A装着B，B可以得到A里面的变量，但A不能得到B里面的变量，那么B就是闭包。
            那么我们需要先补补作用域和作用域链的知识。

            # 作用域

            • 全局作用域
            • 函数作用域
            • 块级作用域（es6新增let,const）简单来说{}就算一个作用域
              看例子：
                var count = 100; //全局作用域
    function foo(){
        var count = 0; //函数作用域
        return count; //返回函数
    }
    if(count==0){
        //块级作用域
        console.log(count)
    }

            # 作用域链

            在作用域里面嵌套作用域，就形成了作用域链。外部作用域无法访问内部作用域
            看例子：

                function foo1(){
        var n=1;
        function foo2(){
            var m=1;
            console.log(n); //1
        }
        foo2();
    }
    foo1();
    console.log(n) //n is not defined

                var name = '前端';

        console.log(name) //输出 前端还是后端?

        var name = '后端';

            答案：前端,why？

            javascript是按照代码书写时候的位置来决定的，而不是按照调用时候的位置。可以按照如下思路理解分析。

            • 调用changeName()时，找到这个函数，然后执行其代码。
            • 定义var name=‘后端’;
            • 调用showName()函数;
            • 在调用changeName函数内部查找是否有showName()这个方法，发现没有，则向外层查找，找到了;
            • 调用console.log(name)方法，在函数内部查找发现没有name,向上一层查找，找到了name=‘前端’;
            • 输出前端

            # 闭包

            好了，重新说回闭包。一开始我们说了概念，那接下来看例子:

                var count = 100; //全局作用域
    function foo(){
        var count = 0; //函数作用域
        function foo1(){
            count++;
            console.log(count);
            return count;
        }
        return foo1;
    }
    let result = foo();
    result(); //count==1
    result(); //count==2
    result(); //？？

            如上所示：在foo1函数内部调用了外部变量count，就形成了闭包。count的值就被保存到了foo1函数的作用域中。所以第二次输出count的值就变成了2.当然过度使用闭包会形成内存泄漏，需要谨慎使用。 再来看一个面试中常见的循环闭包： 1-5每隔一秒依次输出

                for(var i=1;i<5;i++){
        setTimeout(()=>{
            console.log(i)  //6 6 6 6 6
        },1000)
    }

            为什么会输出66666呢？因为setTime是异步执行的，所以等输出，早就循环完了。 可以利用let变成块作用域，这样每一层循环都是独立的，互不影响。

                for(let i=1;i<6;i++){
        setTimeout(()=>{
            console.log(i)  //1 2 3 4 5
        },1000*i)
    }
