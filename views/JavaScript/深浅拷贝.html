              js难搞之深浅拷贝

              小俞儿 2020-05-26 深浅拷贝 JavaScript

              # 前言

              为什么要搞懂深浅拷贝，因为

              • 面试经常会被问到
              • 在工作项目中，涉及到较为复杂点的数据结构也会用到
              • 能好的去理解js的数据类型，内存

              # 栈内存与堆内存

              • js的基本数据类型（String，Number等）保存在栈内存中
              • js的引用数据类型（数组，对象）保存在堆内存中，栈内存中存放着该数据的引用地址

              # 浅拷贝 技巧

              一般深浅拷贝都是基于引用类型的，基本类型只是简单的传值。参考如下代码：

              var a=1;
var b=1;
var c=a;
a === c; //true
b === c; //true
a === b; //true
a = 2;
a === c; //false
b === c; //true
a === b; //false

              # 数组的浅拷贝

              投机取巧，利用数组的一些方法，slice(),concat()返回一个新数组的特性来实现数组的浅拷贝

              var old_arr = ['old_arr',1,null,false,'undefined'];
var new_arr1 = old_arr.concat();
var new_arr2 = old_arr.slice();
new_arr1[0] = 'new_arr';
new_arr2[0] = 'new_arr';
console.log(old_arr); // ["old_arr", 1, null, false, "undefined"]
console.log(new_arr1); // ["new_arr", 1, null, false, "undefined"]
console.log(new_arr2); // ["new_arr", 1, null, false, "undefined"]

              # 概念总结

              如果数组元素是基本数据类型，就会拷贝一分，互不影响，而如果数组元素是对象或者数组，就会只拷贝对象或者数组存放在栈内存中的引用地址，所以无论在新旧数组中做任何修改，两者都会发生改变。我们把这种复制引用的拷贝方法称为浅拷贝。与之对应的就是深度拷贝。神拷贝只完全拷贝一个对象，即使嵌套了对象，两者也相互分离，互不影响。修改任意一个的属性值，也不会改变另外一个。

              # 深拷贝 技巧

              适用于数组和对象，技巧性简单粗暴，注意操作。

                  var old_arr = ['old_arr',1,null,{name:'old'},['old'],false,'undefined'];

    console.log(old_arr); ['old_arr',1,null,{name:'old'},['old'],false,'undefined'];

              # 正规操作实现深浅拷贝。

              浅拷贝

              思路 遍历对象，然后把属性和属性值都放在一个新的对象。

                  var shallowCopy = function(obj){
        // 判断只拷贝对象
        if(typeof obj != 'object'){
            return
        }else{
            // 根据obj类型判断是新建一个数组还是对象
            var newObj = object instanceof Array ? [] : {};
            for(var key in obj){
                if(obj.hasOwnProperty(key)){
                    newObj[key] = obj[key]
                }
            }
            return newObj;
        }
    }

              深拷贝

              深拷贝思路 在浅拷贝的时候判断一下属性值，如果是对象的话，我们就递归调用浅拷贝函数就行了

                  var deepCopy = function(obj){
        // 判断只拷贝对象
        if(typeof obj != 'object'){
            return
        }else{
            // 根据obj类型判断是新建一个数组还是对象
            var newObj = object instanceof Array ? [] : {};
            for(var key in obj){
                if(obj.hasOwnProperty(key)){
                    // 如果属性值是对象递归调用
                    newObj[key] = typeof obj[key] === 'object' ? deepCopy(obj[key]) : obj[key]
                }
            }
            return newObj;
        }
    }

              # 性能问题

              深拷贝会完全克隆一个对象，不会产生副作用，但是深拷贝实现使用递归，性能肯定不如浅拷贝。项目中应该根据情况选择。
