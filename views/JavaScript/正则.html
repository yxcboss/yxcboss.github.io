      这些简单常用的正则你不能不懂

      小俞儿 2020-06-15 正则表达式

      # 何为正则

      正则表达式(regular expression)描述了一种字符串匹配的模式（pattern），可以用来检查一个串是否含有某种子串、将匹配的子串替换或者从某个串中取出符合某个条件的子串等。
      太官方了，不太懂。没事，写博客就是为了自己与他人弄懂。

      # 正则基础

      正则是一种匹配模式，用来匹配字符（普通字符和元字符），或者匹配位置。

      # 匹配字符

      # 精确匹配

      精确匹配没有多大的意义。比如/123456/，就只能匹配123456这串字符。这样就可以直接把输入的值和想要的值对比就可以了。

      var reg = /123456/;
console.log(reg.test(123456));
//true

      # 强大的模糊匹配

      # 横向模糊匹配

      一个正则可以匹配的字符串长度是不固定的，用量词来实现，如{m,n},便是连续最少出现m次，最多出现n次。
      测试如下：

      var reg = /36{2,4}9/g;
var string = '369 3660 36669'
console.log(string.match(reg));
//['3660','36669']
      1
      2
      3
      4

      注意⚠️：g是正则里面的修饰符。表示全局匹配。即在字符串中按顺序找到满足匹配模式的所有字串，强调的是‘所有’，而不是第一个。g为global。

      # 纵向模糊匹配

      一个正则具体匹配到某个字符时，他可以不是某个确定的字符，可以有多种可能，用字符组来实现。如[abc],表示可以是字符‘a’,'b','c'中的任意一个。 测试如下：

      var reg = /a[123]b/g;
var string = 'a1b a2b a3b a4b';
console.log(string.match(reg));
//["a1b", "a2b", "a3b"]

      # 字符组

      # 范围表示法

      当字符组里的字符特别多的话，该怎们办呢？使用范围表示法。
      比如[123456abcdefgHIJKLMN],可以写成[1-6a-gH-N]。用连字符-来省略和简写。
      如何要匹配'a','-','z'着三个字符中的任意一个时，又该怎么写呢？[a-z]表示小写字符中任意一个，显然不行，所以要写成下面这样。
      [-az]或者[az-]或者[a-z]即可。要么放在开头或结尾，要么转义。

      # 排除字符组

      当我们不知道匹配的某个字符是什么，只知道它不能是什么的时候，就要用到排除字符组。例如[^abc],表示一个除了'a','b','c'之外的任意一个字符。字符组的第一个字符为^（脱字符）时，表示求反概念。

      # 常见的简写形式

      • \d就是[0-9]。表示是一位数字。记忆方式：其英文是digit（数字）。
      • \D就是[^0-9]。表示除数字外的任意字符。
      • \w就是[0-9a-zA-Z_]。表示数字、大小写字母和下划线。记忆方式：w是word的简写，也称单词字符。
      • \W是[^0-9a-zA-Z_]。非单词字符。
      • \s是[ \t\v\n\r\f]。表示空白符，包括空格、水平制表符、垂直制表符、换行符、回车符、换页符。记忆方式：s是space character的首字母。
      • \S是[^ \t\v\n\r\f]。 非空白符。
      • .就是[^\n\r\u2028\u2029]。通配符，表示几乎任意字符。换行符、回车符、行分隔符和段分隔符除外。记忆方式：想想省略号...中的每个点，都可以理解成占位符，表示任何类似的东西。
        如果想匹配任意字符呢？就需要配合使用了。
        [\d\D]、[\w\W]、[\s\S]和[^]中任何的一个即可。

      # 量词

      量词也称为重复，即{m,n}重复m到n次之间。

      # 简写形式

      • {m,}表示最少出现m次。
      • {m}等价于{m,m}表示,出现m次。
      • ? 等价与{0,1},表示出现或者不出现。
        • 等价于{1,},表示至少出现1次。
      • *等价于{0,},表现出现任意次，也可不出现。

      # 贪婪匹配于惰性匹配

      先看例子：

      var reg=/\d{2,5}/g;
var string = '123 1234 12345 123456';
console.log(string.match(reg))
// [123,1234,12345,12345]
      1
      2
      3
      4

      分析：正则/\d{2,5}/g，表示匹配连续出现2到5个数字，但是是贪婪的，尽可能多的匹配。最多连续出现5个数字，我就匹配5个连续数字的字符。而惰性匹配就是尽可能少的匹配：

      var reg=/\d{2,5}?/g;
var string = '123 1234 12345 123456';
console.log(string.match(reg))
// ["12", "12", "34", "12", "34", "12", "34", "56"]
      1
      2
      3
      4

      分析：正则/\d{2,5}?/g，表示匹配连续出现2到5个数字，但是后面加了一个问好，则匹配到连续两个数字时就够了，不再继续匹配更多了。

      # 匹配位置

      # 什么是位置

      位置是相邻两个字符之间的位置。如下图

      # ^和$

      ^(脱字符)匹配开头
      $(美元符)匹配结尾
      例如：

      var result = 'hello'.replace(/^|$/g,'#');
console.log(result);
//'#hello#'
      1
      2
      3

      # \b和\B

      \b是单词边界，具体就是\w和\W之间的位置。也包括\w和^,\w和$的位置。
      例子：

      var result = '[JS] Lesson_01.mp4'.replace(/\b/g,'#');
console.log(result);
//[#JS#] #Lesson_01#.#mp4#
      1
      2
      3

      \B是非单词边界，出去\b其他的都是\的。

      var result = '[JS] Lesson_01.mp4'.replace(/\b/g,'#');
console.log(result);
//#[J#S]# L#e#s#s#o#n#_#0#1.m#p#4
      1
      2
      3

      # (?=p)和(?!p)

      （?=p）,其中p是一个自模式，即p前面的位置。
      比如（?=l）,表示l字符前面的位置。

      var result = 'hello'.replace(/(?=l)/g,'#');
console.log(result);
// he#l#lo;
      1
      2
      3

      而(?!p)就是(?=p)的对立面，相反的意思。比如：

      var result = 'hello'.replace(/(?!l)/g,'#');
console.log(result);
// #h#ell#o;

      # 正则常用方法（api）

      # search

      search() 方法用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串。

      var string = 'hello'
var reg = /ll/g;
var reg2 = /qw/g;
string.search(reg);//2
string.search(reg2);//-1

      # match

      match() 方法可在字符串内检索指定的值，或找到一个或多个正则表达式的匹配。

      var str="1 plus 2 equal 3";
var reg = /\d+/g;
str.match(reg);//[1,2,3]

      # replace

      replace() 方法用于在字符串中用一些字符替换另一些字符，或替换一个与正则表达式匹配的子串。

      var str="2020-7-22";
var reg = /-/g;
str.replace(reg,'/');//2020/7/22

      # test

      test() 方法用于检测一个字符串是否匹配某个模式.返回true或者false

      var str="hello world";
var reg = /\d/g;
var reg2 = /world/g;
reg.test(str);//false
reg2.test(str);//true
