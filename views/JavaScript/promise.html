            # 前言

            在异步编程中promise有着不可替代的地位比起传统的解决方法（回调函数）更加合理且更加强大。但不总结每次都是疑问百出，本文让我们一起深入理解这个es6新增API---Promise。

            # 错误处理

            promise对象的错误具有‘冒泡性质’，会一直向后传递，直到被reject函数处理或catch语句捕获为止

            # 链式调用

            • 每次对promise调用then，它都会创建并返回一个promsie对象，我们可以将其链接起来
            • 不管从then调用的完成回调返回值是什么，它都会被自动设置为被链接promise的完成
                    var p1 = new Promise((resolve,reject)=>{
            resolve('success')
        })
        var p2 = p1.then(value=>{
            console.log(value)
        });
        var p3 = p2.then(value=>{

            # Promise.all(arr)

            接受一个promise数组，生成并返会一个Promise对象，可用Promise实例的所有方法 如果参数有任何一个参数为reject时，都会终止Promise.all(arr)的调用

                var p1 = new Promise((resolve,reject)=>{
        resolve('p1')
    })
    var p2 = p1.then(value=>{
        console.log(value); //p1
        return Promise.resolve('p2')
    });
    var p3 = p2.then(value=>{
        console.log(value);//p2
        return Promise.resolve('p3')
    })
    Promise.all([p1,p2,p3]).then(value=>{
        console.log(value) //[p1,p2,p3]
    })

            # Promise.allSettled(arr)

            • 假如有这样的场景：一个页面有三个区域，分别对应三个独立的接口数据，使用 Promise.all 来并发请求三个接口，如果其中任意一个接口出现异常，状态是reject,这会导致页面中该三个区域数据全都无法出来，显然这种状况我们是无法接受，Promise.allSettled的出现就可以解决这个痛点
                Promise.allSettled([
        Promise.reject({ code: 500, msg: '服务异常' }),
        Promise.resolve({ code: 200, list: [] }),
        Promise.resolve({ code: 200, list: [] })
    ]).then(res => {
        console.log(res)
    /*
        0: {status: "rejected", reason: {…}}
        1: {status: "fulfilled", value: {…}}
        2: {status: "fulfilled", value: {…}}
    */
    // 过滤掉 rejected 状态，尽可能多的保证页面区域数据渲染
    var newres = res.filter(el => {
      return el.status !== 'rejected'
    })
    console.log(newres)
    /*
        1: {status: "fulfilled", value: {…}}
        2: {status: "fulfilled", value: {…}}
    */
})

            1. Promise.prototype.finally()

            ES9 新增 finally() 方法返回一个Promise。在promise结束时，无论结果是fulfilled或者是rejected，都会执行指定的回调函数。这为在Promise是否成功完成后都需要执行的代码提供了一种方式。这避免了同样的语句需要在then()和catch()中各写一次的情况。

                //比如我们发送请求之前会出现一个loading，当我们请求发送完成之后，不管请求有没有出错，我们都希望关掉这个loading。
    this.loading = true
    request()
    .then((res) => {
        // do something
    })
    .catch(() => {
        // log err
    })
    .finally(() => {
        this.loading = false
    })

            # async & await 实现

            es7新增异步编程方法，基于promise，async返回promise对象，是genterator的语法糖。

            • 语法简洁，更像是同步代码，符合阅读习惯
            • 改进js中异步操作then的链式调用，减少callback的嵌套
                let p1 = new Promise(resolve=>{
            resolve(1)
        })
        let p2 = new Promise((resolve,reject)=>{
            setTimeout(()=>{
                resolve(2)
                // reject(2) 不继续运行
            },2000)
        })
        async function fn(){
            console.log(1)
            let result1 = await p2;
            console.log(3)//等待2秒后输出3
            let result2 = await p1;
            console.log(4)
        }
        fn();
        console.log(2)
        //1,2,3,4
